Take-Home Challenge — Reflex Card Game
Build a two-player reflex game where cards appear one by one. When an Ace appears, the first player
who reacts (clicks) wins the round. The goal of this challenge is to evaluate how you design and build a
small but complete interactive product, including system structure, real-time interaction, and
deployment. A fully-fledged multiplayer experience where two players can join from different locations
over the internet is strongly encouraged.
Game Rules
1 Two players join the same game.
2 Cards appear sequentially on the screen.
3 Players must wait until an Ace appears.
4 When an Ace appears, the first player to click wins.
5 If a player clicks before an Ace appears, they lose the round.
Expectations
• Treat this as a small production-quality product, not just a coding exercise.
• You are free to choose the technology stack, architecture, UI/UX design, real-time approach, and
deployment strategy.
• Provide a working two-player experience.
• Write clear and maintainable code.
• Provide a live deployed version of the game.
• Include a short explanation of your technical decisions.
AI Usage
You are encouraged to use AI tools such as Claude, ChatGPT, Cursor, Copilot, or similar tools. During
the interview we may ask how you used AI while building the project, what parts were AI-generated,
and how you validated or modified the output.
Deliverables
• GitHub repository containing the source code and instructions to run locally.
• A live deployment with a public URL where the game can be played.
• A short README explaining the architecture, key design decisions, and tradeoffs.
